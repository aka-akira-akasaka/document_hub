## プロジェクト概要

**目的：ソリューションの販売パートナー開拓と、開拓後の活動数最大化を実現するためのパワーチャートを管理する**

販売パートナー（代理店・販売店・紹介パートナー等）の獲得から活動推進まで、**組織力学と担当者レベルのポジションの可視化**をコア機能として両フェーズを支援するSaaSツール。

```
【フェーズ1：開拓】パートナー候補組織の内部を可視化し、キーマンを攻略してパートナー化する
         ↓
【フェーズ2：活動最大化】パートナー化後、組織内の力学を継続把握し「誰に働きかければ活動が増えるか」を特定する
```

> **SFAとの棲み分け**
> 活動数の記録・集計・目標管理はSFA（営業支援システム）の領域。
> 本ツールは「**なぜ活動が増えない／増えるのか**」を組織図・人物ポジションで説明するための戦略ツールとして位置づける。

| 項目 | 内容 |
|---|---|
| 対象ユーザー | チャネル営業担当者・販売パートナー開拓マネージャー |
| コア機能 | 組織力学と担当者レベルのポジションの可視化 |
| フェーズ1の価値 | パートナー候補組織のキーマン特定・攻略戦略の立案 |
| フェーズ2の価値 | パートナー組織内の推進者・阻害者を特定し、アプローチ先を明確化 |
| 現フェーズ | フロントエンドMVP+ （LocalStorage + Supabaseバックアップ） |
| リポジトリ | `projects/saas-power-chart/` |

## データモデル

### Deal（開拓案件）

パートナー候補・パートナー企業1社 = 1 Deal。開拓前〜パートナー化後を通じて組織図・ポジション情報を継続管理する。

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | UUID |
| name | string | 案件名（例：「◯◯商事 パートナー化」） |
| clientName | string | パートナー候補・パートナー企業名 |
| stage | DealStage | 開拓ステージ（7段階） |
| description | string | 概要 |
| targetAmount | number? | 目標取扱高 |
| expectedCloseDate | string? | パートナー契約期待日 |

**DealStage（現在の実装）:**
`prospecting`（初期接触）→ `qualification`（条件確認）→ `proposal`（提案中）→ `negotiation`（交渉中）→ `closed_won`（パートナー契約締結）/ `closed_lost`（開拓断念）/ `on_hold`（保留）

> ⚠️ **要検討（B-25）:** 現ステージは汎用商談フローのまま。パートナー開拓フローに合わせた再定義を検討中。

### Stakeholder（ステークホルダー）

パートナー候補・パートナー企業の内部関係者。フェーズを問わず「組織内でどのポジションにいるか」を継続的に管理する。

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | UUID |
| dealId | string | 紐づく案件ID |
| name | string | 氏名 |
| department | string | 部署（groupIdから自動導出） |
| title | string | 役職（orgLevelから対応） |
| roleInDeal | RoleInDeal | 案件上の役割（8種） |
| influenceLevel | 1〜5 | 影響力レベル |
| attitude | Attitude | パートナー化・活動への態度（5種） |
| relationshipOwner | string | 担当者（自社側） |
| parentId | string? | 上位ノードID（組織図階層） |
| groupId | string? | 所属部署グループID |
| orgLevel | number | 役職階層レベル |
| email / phone / notes | string | 連絡先・メモ |
| mission | string | ミッション |
| position | {x,y}? | フリーフローティング時の座標 |

**RoleInDeal:** 意思決定者 / 承認者 / 起案者 / 評価者 / ユーザー / ゲートキーパー / 不明
**Attitude:** チャンピオン（推進）/ 支持者 / 中立 / 慎重 / 反対者
— フェーズ1では「パートナー化への協力度」、フェーズ2では「パートナー活動の推進・阻害度」として同じ軸を読み替えて使用する。

### OrgGroup（部署グループ）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | UUID |
| dealId | string | 紐づく案件ID |
| name | string | 部署名 |
| parentGroupId | string? | 親部署ID（階層ネスト） |
| color | string? | 表示色 |
| sortOrder | number | 並び順（D&D対応） |

### Relationship（関係性）

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | UUID |
| dealId | string | 紐づく案件ID |
| sourceId / targetId | string | 関係元・先のステークホルダーID |
| type | RelationshipType | 関係種別（6種） |
| label | string? | エッジラベル |
| bidirectional | boolean | 双方向フラグ |
| color | string? | エッジ色 |

**RelationshipType:** 上下関係 / 影響関係 / 同盟関係 / 対立関係 / 非公式 / 監督（人→グループ）
