# projects/ — プロジェクトディレクトリ

> 開発本部（DevHQ）が管轄する全てのコードプロジェクトを格納するディレクトリ。

---

## ディレクトリ構成

```
projects/
├── README.md               # このファイル
├── <プロジェクト名>/        # 各プロジェクト
│   ├── README.md           # プロジェクト概要・セットアップ
│   ├── src/                # ソースコード
│   ├── tests/              # テスト
│   └── ...
```

## プロジェクト命名規則

- 英語のケバブケース: `bank-demo-app`, `internal-report-tool`
- プレフィックスで種別を示す:
  - `saas-*` — SaaSプロダクト
  - `tool-*` — 社内ツール
  - `poc-*` — PoC・プロトタイプ
  - `lp-*` — ランディングページ

## 新規プロジェクト作成時のチェックリスト

- [ ] 赤坂さんにスコープ・目的を承認してもらった
- [ ] README.md にプロジェクト概要・セットアップ手順を記載
- [ ] .env.example に必要な環境変数を列挙（秘匿値は含めない）
- [ ] .gitignore を適切に設定（node_modules, .env, etc.）
