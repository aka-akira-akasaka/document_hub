# Claude Code × モダンSaaSスタック 実践ガイド

> GitHub + Vercel + Supabase + Claude Code で、個人でもSaaSが作れる時代

---

## はじめに

本ドキュメントは、Power Chart（エンタープライズ組織パワーチャート作成ツール）の開発実例をもとに、**Claude Code を活用したモダンSaaS開発**の全体像と具体的な手順を解説します。

### この構成で実現できたこと

- Google認証付きマルチユーザーSaaS
- ユーザー別データ分離（Row Level Security）
- 自動デプロイ（git push → 本番反映）
- **開発期間: 1セッション（数時間）**
- **インフラ費用: 0円**（全サービス無料枠内）

## 各サービスの役割と選定理由

### 1. GitHub — ソースコード管理

| 項目 | 内容 |
|------|------|
| 役割 | バージョン管理、Vercelとの連携トリガー |
| 無料枠 | 無制限（パブリック/プライベート） |
| 選定理由 | Vercelとのネイティブ連携、業界標準 |

### 2. Vercel — ホスティング & CI/CD

| 項目 | 内容 |
|------|------|
| 役割 | Next.jsアプリのビルド・ホスティング・自動デプロイ |
| 無料枠 | Hobby プラン（個人利用無料、商用は Pro $20/月） |
| 選定理由 | Next.jsの開発元、ゼロコンフィグデプロイ |

**特筆すべき点:**
- `git push origin main` → 自動で本番デプロイ（所要時間 約1分）
- ブランチごとにプレビューURLが自動生成される
- 環境変数はダッシュボードで管理

### 3. Supabase — 認証 & データベース

| 項目 | 内容 |
|------|------|
| 役割 | Google OAuth認証、PostgreSQLデータベース、Row Level Security |
| 無料枠 | 2プロジェクト、500MB DB、50,000 MAU |
| 選定理由 | Firebase代替のOSS、SQLが使える、RLSでセキュリティ担保 |

**特筆すべき点:**
- Google / GitHub / Microsoft 等の OAuth プロバイダーを数クリックで有効化
- Row Level Security (RLS) により、DBレベルでユーザー間のデータを完全分離
- REST API が自動生成される（バックエンドコード不要）

### 4. Claude Code — AI開発パートナー

| 項目 | 内容 |
|------|------|
| 役割 | 設計・実装・デバッグ・デプロイ支援の全工程 |
| 特徴 | コードベース全体を理解した上での実装、対話的な設計判断 |

## Claude Code が担った役割（実例）

### Power Chart での実際の作業フロー

```
1. 要件の整理
   ユーザー: 「社内アーリーアダプターにGoogle IDでログインして使えるようにしたい」

2. 実装（12ファイル新規作成 + 6ファイル修正）
   - DBスキーマ設計 (SQL)
   - 認証フロー実装 (OAuth callback, middleware, AuthProvider)
   - データ同期エンジン (Zustand ↔ Supabase)
   - UI改修 (ログインページ, ユーザーメニュー)
   → 既存の20+コンポーネントは一切変更なし

3. 外部サービス設定のガイド
   - Supabase プロジェクト作成手順
   - Google Cloud Console OAuth設定手順

   → スクリーンショットを見ながら対話的にサポート

4. テスト & デプロイ
   - ローカル動作確認
   - 本番デプロイ
   - マルチユーザーでのデータ分離確認
```

| 観点 | 具体例 |

### Phase 1: プロジェクト作成（Claude Code）

```bash

「Next.js + Zustand でパワーチャート作成ツールを作って」

# - UIコンポーネント実装
# - 状態管理 (Zustand) 設計・実装
# - ローカルで動作確認
```

### Phase 2: 認証 & DB 追加（Claude Code + ユーザー）

```bash
# Claude Code に依頼
「Google認証でマルチユーザー対応にして」

# Claude Code が実行すること:
# - Supabase スキーマ設計 (SQL生成)
# - 認証フロー実装
# - データ同期エンジン実装
# - 既存ストアの改修（最小限）

# ユーザーが実行すること（Claude Codeがガイド）:
# - Supabase プロジェクト作成
# - Google Cloud Console でOAuth設定
# - 環境変数の設定
```

### Phase 3: デプロイ（Claude Code + ユーザー）

```bash
# Claude Code が実行すること:
# - git commit & push

# ユーザーが実行すること（Claude Codeがガイド）:
# - Vercel でプロジェクト作成（GitHubリポジトリ連携）
# - 環境変数設定
# - Supabase / GCP に本番URLを追加

# 以降のデプロイ:
git push origin main  # これだけで自動デプロイ
```

## よくある質問

### Q: セキュリティは大丈夫？

**A:** Supabase の Row Level Security (RLS) により、DBレベルでユーザー間のデータが完全に分離されます。認証トークンの検証も Supabase Auth が自動で行います。APIキーが漏洩しても、RLSにより他ユーザーのデータにはアクセスできません。

### Q: スケールできる？

**A:** Vercel は CDN ベースで自動スケール、Supabase の PostgreSQL も負荷に応じてスケール可能です。数万ユーザーまでは無料〜低コストで運用できます。

### Q: Claude Code なしでも保守できる？

**A:** 生成されるコードは標準的な Next.js + TypeScript なので、通常のエンジニアが保守可能です。Claude Code は魔法の独自コードを生成するわけではなく、ベストプラクティスに沿った実装を行います。

### Q: 既存の社内ツールにも適用できる？

**A:** はい。特に以下のようなケースに適しています：
- 社内で使っているExcel管理ツールのWeb化
- 部門間の情報共有ダッシュボード
- 顧客管理・案件管理ツール
- 承認ワークフロー
